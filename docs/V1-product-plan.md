# English App V1 产品规划文档

## 1. 文档目的

本文档用于明确 `English App` 的 V1 产品方向，解决“项目已经跑通，但产品形态仍然模糊”的问题。

这份文档的目标不是追求大而全，而是帮助项目在现阶段形成一个清晰、可执行、可闭环的英语学习产品方案，指导后续的：

- 前端页面扩展
- 后端接口设计
- 数据库结构迭代
- 版本开发优先级安排

---

## 2. 产品一句话定义

`English App` 是一个面向日常英语提升场景的轻量级英语学习网站，帮助用户通过“每日任务 + 场景课程 + 单词记忆 + 学习反馈”形成持续学习闭环。

---

## 3. V1 产品定位

### 3.1 我们要做的，不是资料站

V1 不应该只是“课程展示页 + 单词展示页 + 登录页”的拼接，而应该是一个可以每天使用的学习产品。

用户进入网站后，应该明确知道：

- 今天学什么
- 从哪里开始
- 学完后会得到什么反馈
- 明天还能怎么继续

### 3.2 V1 的核心定位

V1 建议定位为：

“一个帮助英语学习者建立日常学习习惯的场景化英语学习平台。”

关键词：

- 场景化：围绕旅行、职场、日常交流等真实使用场景
- 轻量化：每次学习 15 到 25 分钟
- 持续化：强调连续学习和进步反馈
- 闭环化：内容、练习、记录、反馈串起来

---

## 4. 目标用户

### 4.1 核心用户画像

V1 主要服务以下人群：

- 有英语基础，但长期缺乏稳定学习节奏的大学生
- 工作后想重新提升英语表达能力的职场人
- 希望通过碎片化时间积累词汇和表达的自学者

### 4.2 用户共性特征

- 想学，但容易三天打鱼两天晒网
- 不知道每天该学什么
- 看过很多资料，但缺少持续执行机制
- 希望看到可量化的进步，而不是只“感觉自己学了”

### 4.3 V1 不重点覆盖的人群

- 备考深度型用户，如雅思托福高阶专项刷题用户
- 强依赖直播、社群、老师陪伴的用户
- 需要完整语法课程体系的零基础启蒙用户

---

## 5. 用户核心问题

V1 要解决的核心问题不是“用户没有英语资料”，而是下面这几个问题：

1. 用户不知道每天该学什么
2. 用户学习内容零散，缺少结构
3. 用户学完后没有即时反馈，难以坚持
4. 用户无法清楚看到自己的积累与成长

因此，V1 的重点不是内容规模，而是学习路径和反馈闭环。

---

## 6. V1 产品目标

### 6.1 产品目标

在 V1 阶段，产品需要完成三件事：

1. 帮用户建立“每日进入学习”的理由
2. 帮用户完成一次完整学习流程
3. 帮用户看到结果，并愿意第二天继续回来

### 6.2 业务目标

V1 的核心不是商业化，而是先验证产品逻辑是否成立：

- 用户是否愿意完成每日学习任务
- 用户是否能理解产品结构
- 用户是否能感知自己的进步
- 项目是否具备继续扩展成完整学习平台的基础

---

## 7. 产品核心闭环

V1 的核心闭环建议定义为：

1. 用户登录产品
2. 首页看到今日学习任务
3. 进入课程完成一次学习
4. 进入单词页完成今日单词记忆
5. 系统记录学习行为
6. 数据页/首页展示学习反馈
7. 次日继续学习新的内容或复习内容

这个闭环意味着：

- 首页负责“告诉用户今天做什么”
- 课程页负责“输入内容”
- 单词页负责“巩固记忆”
- 数据页负责“输出反馈”
- 用户系统负责“承载身份和进度”

只要这条链路顺畅，产品就已经具备了 V1 的基本完成度。

---

## 8. V1 核心价值主张

对用户来说，V1 的价值不是“内容很多”，而是：

- 每天都能快速开始学习
- 学习内容围绕真实场景，不空泛
- 单词与课程关联，记忆更自然
- 学习后能看到清晰反馈，增强成就感

---

## 9. V1 功能范围

### 9.1 必做功能

以下功能建议定义为 V1 范围内的必须项。

#### A. 用户系统

- 注册
- 登录
- JWT 鉴权
- 获取当前用户信息
- 退出登录

#### B. 首页

- 展示今日学习概览
- 展示推荐课程
- 展示今日单词
- 展示学习建议
- 提供继续学习入口

#### C. 课程系统

- 课程列表
- 课程详情
- 展示课程难度、时长、课时数、简介
- 支持标记“开始学习”或“完成课程”

#### D. 单词系统

- 今日单词列表
- 展示单词、音标、释义、例句、分类、难度
- 支持“已掌握 / 未掌握”状态
- 支持关联到用户学习进度

#### E. 学习记录系统

- 记录用户每日学习时长
- 记录用户学习单词数
- 记录用户完成课程数
- 支持首页和统计页读取这些数据

#### F. 数据反馈系统

- 首页展示今日学习时长、累计词汇、本周课程数
- 统计页展示近 7 天学习趋势
- 展示成长路径或阶段建议

### 9.2 建议纳入 V1.1 的功能

这些功能很有价值，但不一定要在 V1 首发时全部完成：

- 课程章节拆分
- 单词收藏
- 错词本
- 每日打卡
- 连续学习天数
- 简单测验
- 学习目标设置

### 9.3 暂不纳入 V1 的功能

以下内容先不建议在当前阶段投入：

- AI 口语陪练
- 作文批改
- 音视频上传
- 完整后台管理系统
- 复杂题库系统
- 会员付费体系
- 社区、评论、好友排名

---

## 10. V1 最小可行闭环

为了确保项目可交付，建议把 V1 的最小闭环控制在下面这个范围：

1. 用户注册/登录成功
2. 登录后首页看到自己的学习概览
3. 用户进入课程页选择课程
4. 用户进入单词页学习今日单词
5. 用户的学习记录被保存
6. 数据页能看到本周学习变化

这个版本虽然不复杂，但已经能够回答“这是一个什么产品”。

## 11. 页面结构规划

### 11.1 页面清单

V1 建议保留以下页面：

- `/auth`
  - 登录
  - 注册

- `/`
  - 首页
  - 今日学习概览
  - 推荐课程
  - 今日单词
  - 学习建议

- `/courses`
  - 课程列表

- `/courses/:id`
  - 课程详情
  - 课程信息
  - 课程目标
  - 课程学习入口

- `/words`
  - 今日单词
  - 已掌握/未掌握状态

- `/stats`
  - 近 7 天学习统计
  - 成长路径

- `/profile`
  - 用户资料
  - 学习目标
  - 退出登录

### 11.2 页面导航关系

建议主导航保持简单：

- 首页
- 课程
- 单词
- 数据
- 我的

## 12. 用户核心使用路径

### 12.1 新用户路径

1. 打开首页
2. 浏览课程和产品信息
3. 注册账号
4. 完成首次登录
5. 进入首页看到推荐课程和今日单词
6. 开始第一次学习

### 12.2 老用户日常路径

1. 登录产品
2. 查看首页今日任务
3. 学习课程
4. 完成今日单词
5. 查看学习反馈
6. 退出，第二天继续

---

## 13. 当前项目能力与 V1 对照

### 13.1 当前已经具备的能力

根据当前代码结构，项目已具备以下基础：

- 前后端项目架构已搭建并跑通
- 用户注册、登录、JWT 鉴权已接入
- 首页、课程页、单词页、数据页基础页面已存在
- 后端已有课程、今日单词、学习看板接口
- 前端已完成 React + TypeScript + Tailwind CSS 基础建设

### 13.2 当前仍缺少的关键能力

要达到真正的 V1 闭环，还需要补齐：

- 课程详情页
- 课程完成状态记录
- 单词掌握状态记录
- 用户个人学习进度存储
- 统计页真实接口化，而不是静态 mock 数据
- 用户资料页

### 13.3 当前最适合的推进策略

不是继续横向堆功能，而是优先把已有模块串起来：

1. 补进度数据
2. 补课程详情
3. 补单词状态
4. 补统计接口
5. 补个人中心

## 14. 核心数据模型建议

结合当前项目结构，V1 建议至少围绕以下实体扩展。

### 14.1 用户

- `app_user`
  - id
  - nickname
  - email
  - password

### 14.2 课程

- `course`
  - id
  - title
  - level
  - description
  - lesson_count
  - duration_minutes
  - cover_url

### 14.3 单词

- `word_book`
  - id
  - word
  - phonetic
  - meaning
  - example_sentence
  - category
  - difficulty

### 14.4 学习记录

- `study_record`
  - id
  - user_id
  - study_date
  - minutes_spent
  - words_learned
  - lessons_completed

### 14.5 建议新增的用户课程进度表

- `user_course_progress`
  - id
  - user_id
  - course_id
  - status
  - progress_percent
  - last_studied_at
  - completed_at

### 14.6 建议新增的用户单词进度表

- `user_word_progress`
  - id
  - user_id
  - word_id
  - status
  - review_count
  - last_reviewed_at

这两张表是形成学习闭环的重要基础，因为“课程”和“单词”只有和“用户进度”绑定，产品才真正具备成长性。

## 16. V1 版本里程碑建议

### 阶段 1：基础闭环打通

目标：把“注册登录 -> 首页 -> 课程 -> 单词 -> 数据反馈”这条主链路打通。

建议事项：

- 完善用户认证体验
- 首页读取真实数据
- 课程列表与单词列表稳定可用
- 数据页改为真实接口驱动

### 阶段 2：学习进度接入

目标：让用户行为可以被系统记录。

建议事项：

- 增加课程进度表
- 增加单词进度表
- 增加课程完成接口
- 增加单词掌握接口

### 阶段 3：课程详情与个人中心

目标：补齐产品基本使用面。

建议事项：

- 增加课程详情页
- 增加个人中心页
- 增加学习目标展示

### 阶段 4：V1 收口与优化

目标：形成一个可以展示和继续迭代的稳定版本。

建议事项：

- 优化页面状态与异常提示
- 完善接口错误处理
- 优化移动端适配
- 补基础测试

---

## 17. V1 成功标准

如果满足下面这些条件，就可以认为 V1 是合理且成立的：

1. 用户可以完成注册、登录、进入产品
2. 用户可以找到当天要学习的内容
3. 用户可以完成至少一次课程学习和一次单词学习
4. 用户的学习行为可以被记录
5. 用户能在首页或统计页看到反馈
6. 产品结构清晰，用户知道下一步该做什么

---

## 18. 产品边界判断原则

后续新增功能时，建议始终用下面三个问题来判断是否应该做：

1. 这个功能是否服务于核心闭环
2. 这个功能是否提升用户持续学习的可能性
3. 这个功能是否适合当前 V1 阶段的投入产出比

如果一个功能回答不了这三个问题，建议先不做。

---

## 19. 结论

`English App` 的 V1 不需要一开始做成复杂的大型英语平台。

更合理的方向是先做成一个“可每天使用的场景化英语学习产品”，让用户能够：

- 登录
- 开始学习
- 记住内容
- 看到反馈
- 持续回来

这才是 V1 最重要的价值。

在当前项目基础上，产品方向已经具备雏形，后续重点不是继续横向堆功能，而是把用户系统、课程、单词、进度、反馈这几块真正串成一个闭环。
